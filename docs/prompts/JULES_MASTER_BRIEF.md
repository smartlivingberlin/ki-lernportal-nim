# Jules Master Brief

You are supporting the development of **KI-Lernportal NIM**, an adaptive AI learning portal for non-technical users.

## Mission

Build a beginner-friendly, accessible, DSGVO-aware portal that helps people understand AI, digital tools, robotics, RAG, agents and NVIDIA-NIM-inspired model categories. The portal must use gentle language, supportive microcopy, clear navigation, and adaptive learning paths.

## Current repository state

The repository contains:

- a static HTML prototype,
- a development plan,
- a NVIDIA/NIM analysis report,
- early presentation material,
- agent working rules.

## Important constraints

- Do not use paid cloud services.
- Do not add API keys.
- Do not commit secrets.
- Do not claim all NVIDIA models were fully validated.
- Keep changes small and reviewable.
- Prefer accessible UI and simple language.
- All PRs must include clear explanation and test instructions.
- Do not merge anything.

## Working style

1. First understand the current documentation.
2. Propose a plan before modifying code.
3. Keep each PR small.
4. Add tests where meaningful.
5. Never merge anything.
6. Human review is required before main branch changes.

## First task should be read-only

Before any code changes, perform a structured architecture audit:

1. Current project status.
2. Missing architecture pieces.
3. Recommended MVP scope.
4. Recommended repo structure.
5. Security/privacy/accessibility risks.
6. Suggested first 10 GitHub issues.
7. Suggested phased roadmap.
8. What Jules should build first and what should remain human-reviewed.
